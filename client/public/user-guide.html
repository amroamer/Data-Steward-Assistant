Introduction

What Is the ZATCA Data Owner Agent?

The ZATCA Data Owner Agent is an AI-powered chat application designed for data governance professionals. It connects to Claude (Anthropic's AI model) and lets you upload a data file — Excel, PDF, Word, or image — describe what you need in plain language, and receive structured, downloadable outputs aligned with Saudi SDAIA NDMO regulations and international data governance standards.

It is built for data owners, data stewards, and data governance teams who need to quickly produce compliant documentation, quality rules, and analysis without manual effort.

Key principle: Upload a file, type your request in plain English or Arabic, and download a professional Excel report — no technical setup required.
Interface

The 3-Panel Command Center

The application is divided into three panels that work together as your AI command center.

Left Panel

Sidebar

ZATCA branding, real-time agent status (Idle / Thinking / Executing / Done), full list of past sessions, and the New Session button.

Center Panel

Work Area

Capabilities dashboard on first load. Active conversations show as collapsible command cards with live progress tracking. Command console at the bottom.

Right Panel

Outputs & Activity

Live result.xlsx download button, color-coded sheet tracker, and a chronological activity timeline for the current session.

Command Console

Located at the bottom of the center panel. Type your request in the text field, attach a file using the paperclip icon, switch between English and Arabic using the language toggle, then click Execute.

Agent Status

The status pill in the top-left sidebar updates in real time:

  • Idle — waiting for your command
  • Thinking — reading your request and planning the analysis
  • Executing — running the AI analysis
  • Done — analysis complete, results saved

Feature 1

🎯 Data Classification

Assigns every field in your dataset a sensitivity classification level following Saudi SDAIA NDMO data governance standards.

How to trigger it

give me data classification classify each field classify data fields data classification for all fields

What you get

  • Sensitivity level for each field: Public, Internal, Confidential, Restricted, or Top Secret
  • Justification explaining why each level was assigned
  • Regulatory notes linking to relevant NDMO guidelines

Output format

A formatted table in the center thread card, plus a Classification sheet added to result.xlsx.

Tip: Upload your Excel or PDF file first, then ask for classification. The agent will analyze every field in the file automatically.
Feature 2

📚 Business Definitions

Generates clear, professional business definitions for every field in your dataset — turning technical column names into documented business terminology.

How to trigger it

give me business definitions define all fields business definitions for all fields generate business definitions

What you get per field

  • Business Name — a plain-language label (not the raw column name)
  • Definition — a full sentence describing what the field represents in a business context
  • Data Type and expected format
  • Example values to illustrate the field
  • Business rules — any known constraints or validation logic

Output format

A formatted table in the center thread, plus a Business Definitions sheet in result.xlsx.

Feature 3

✅ Data Quality Rules — 4 Layers

The most comprehensive feature. The agent generates a complete set of data quality rules covering four distinct layers, using two sequential AI calls to ensure full depth without token limits.

How to trigger it

give me data quality rules generate DQ rules data quality rules for all fields quality check validation rules

The 4 Layers

Layer 1

Technical Rules

Nullability, data type validation, maximum length, encoding format, date format patterns.

Layer 2

Logical Rules

Range checks, valid value lists, regular expression patterns, referential integrity.

Layer 3

Business Rules

Domain-specific validity, ZATCA compliance rules, regulatory constraints, business logic.

Layer 4

Cross-Field & Logic Warnings

Rules spanning multiple fields, conditional dependencies, data design warnings.

How the 2-Call Split works

To handle large datasets (30+ fields), the agent splits the analysis into two AI calls that run one after the other. You will see "Part 1/2..." then "Part 2/2..." progress indicators in the center panel. The results are automatically merged and presented as one complete report.

Output format

  • A donut chart showing rule counts by severity (Critical / High / Medium / Low)
  • A summary card with totals by rule type
  • Three sheets in result.xlsx: Technical Rules, Business Rules, Cross-Field Rules
No file? If you ask for DQ rules without uploading a new file, the agent automatically extracts field names from your previous conversation to generate the rules.
Feature 4

⭐ Analytical Data Model

Designs a dimensional (star schema) data model from your dataset, complete with an interactive diagram and ready-to-run DDL SQL scripts.

How to trigger it

give me an analytical data model build a star schema create a dimensional model analytical model for my data

What you get

  • Fact tables identified from business measures and transactional data
  • Dimension tables for descriptive attributes (time, location, customer, product, etc.)
  • Relationships between tables with cardinality defined
  • Interactive SVG diagram — a clickable star schema rendered directly in the thread card
  • DDL SQL script — full CREATE TABLE statements with primary keys, foreign keys, data types, and comments

Output format

The diagram is shown in the center panel. A Data Model sheet is added to result.xlsx, and you can download the DDL script as a separate .sql file directly from the card.

Feature 5

🔒 PII Detection (PDPL Compliance)

Scans every field in your dataset against Saudi Arabia's Personal Data Protection Law (PDPL) and international PII standards to identify privacy risks and compliance gaps.

How to trigger it

detect PII run PII scan scan for sensitive data PII analysis find personal data PDPL check

What you get per field

  • PII Classification: PII / Sensitive PII / Non-PII
  • PII Category: Name, National ID, Financial, Health, Location, Biometric, Contact, etc.
  • Risk Level: Critical / High / Medium / Low
  • PDPL Article reference where applicable
  • Compliance recommendation — encryption, masking, access control, retention policy

Output format

A scan summary card showing total PII fields found by category, plus a PII Scan sheet in result.xlsx.

Feature 6

📊 Data Insights Report

Profiles your uploaded dataset column by column and produces a comprehensive analytical report with executive summary, key findings, and actionable recommendations.

How to trigger it

generate data insight report give me insights analyze this data data insights insights on this data data report

What the report contains

  • Executive Summary — total rows, columns, overall completeness percentage, high-level narrative
  • Key Insights — up to 10 findings covering distributions, anomalies, correlations, trends, and data quality issues
  • Recommendations — up to 8 actionable suggestions, each with a priority (High / Medium / Low) and effort estimate
  • Column Profiles — per-column statistics: min/max, mean, null rate, unique count, top values
  • Data Quality Flags — automated warnings for missing data, outliers, duplicates, format inconsistencies

Output format

A standalone, fully styled Excel report — separate from result.xlsx — with all sections organized across multiple sheets. Download it directly from the agent response card.

Important: This feature requires a file upload in the same message. Attach your Excel, PDF, or CSV alongside your insights request.
File Support

Supported File Formats

Attach any of the following file types to your command using the paperclip icon in the command console.

Format Extensions How it's processed
Excel .xlsx, .xls Field names and data rows are extracted. Column statistics are profiled for the Insights Report.
CSV .csv Parsed as tabular data — same processing as Excel.
PDF .pdf Text is extracted. Table structures are detected where possible.
Word .docx, .doc Text is extracted and treated the same as PDF.
Image .png, .jpg, .jpeg, .gif, .webp Claude's vision model reads the image directly. Files over 4.5 MB are auto-compressed before sending.
Result Management

Cumulative result.xlsx

You can run multiple analyses in a single session. Each completed analysis adds a new sheet to the same result.xlsx file. For example:

  1. Upload your file and ask for Data Classification → Classification sheet added
  2. Ask for Business Definitions → Business Definitions sheet added
  3. Ask for DQ Rules → Three DQ sheets added
  4. Ask for PII Detection → PII Scan sheet added

One download from the right-hand Outputs panel gives you everything from the session in a single organized file.

Sheet Tracker

The right panel shows color-coded tags for every sheet currently inside result.xlsx, so you can see at a glance what's been generated.

Starting a new file: If you upload a new file after already running analyses, the agent will prompt you to confirm — keeping your existing results or resetting and starting fresh with the new file.
Localization

English and Arabic Support

The entire interface supports both English (left-to-right) and Arabic (right-to-left). Click the language toggle button in the command console to switch.

  • All UI labels, buttons, and messages switch to the selected language
  • The 3-panel layout flips direction for proper RTL display in Arabic
  • The AI understands requests written in Arabic and responds accordingly
  • You can also type trigger phrases in Arabic (e.g., رؤى البيانات, تصنيف البيانات)
Mobile Use

Mobile & Touch Devices

The application adapts automatically on smaller screens:

  • The left sidebar becomes a slide-in drawer, accessible from a menu button
  • The right outputs panel becomes a slide-in drawer accessible from the bottom
  • A camera capture button appears on touch devices — tap it to photograph a document or spreadsheet and send it directly for analysis
