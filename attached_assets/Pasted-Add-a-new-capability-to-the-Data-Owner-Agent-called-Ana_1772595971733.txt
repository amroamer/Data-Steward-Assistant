What it does:
When the user uploads an Excel sheet with data fields and asks for an analytical data model, the app should use Claude to analyze the fields and automatically design a star schema / dimensional model, then render it visually inside the chat as an interactive diagram.

Detailed Behavior:
1. Claude Analysis
Send the fields to Claude with a prompt that instructs it to:

Group fields into Fact Tables and Dimension Tables based on their nature (measures vs. descriptive attributes)
Identify primary keys and foreign keys
Suggest grain of each fact table (what one row represents)
Suggest relationships between tables (one-to-many, many-to-many)
Suggest aggregation type for each measure (SUM, COUNT, AVG, etc.)
Return the result as structured JSON only — no prose

The JSON schema should look like:
json{
  "model_name": "...",
  "grain_statement": "...",
  "fact_tables": [
    {
      "table_name": "...",
      "grain": "...",
      "fields": [
        { "field_name": "...", "data_type": "...", "role": "measure|fk|pk", "aggregation": "..." }
      ]
    }
  ],
  "dimension_tables": [
    {
      "table_name": "...",
      "fields": [
        { "field_name": "...", "data_type": "...", "role": "pk|attribute" }
      ]
    }
  ],
  "relationships": [
    { "from_table": "...", "from_field": "...", "to_table": "...", "to_field": "...", "type": "many-to-one" }
  ]
}

2. Visual Rendering
Render the data model visually inside the chat using a canvas or SVG-based diagram (use React state + SVG, or integrate reactflow if available):

Fact tables rendered as center nodes in dark KPMG blue with a bold header
Dimension tables rendered as surrounding nodes in KPMG light blue/grey
Relationship lines drawn between tables with crow's foot notation or simple labeled arrows showing cardinality

Tables should be draggable so the user can rearrange the layout
The diagram should fit inside the chat bubble with a scrollable/zoomable container


3. Export Options
Add two export buttons below the diagram:

"Download as PNG" — exports the diagram as an image
"Download DDL Script" — generates and downloads a .sql file with CREATE TABLE statements, primary keys, foreign keys, and comments for each table based on the model


4. result.xlsx Integration
Also merge the data model output into the cumulative result.xlsx:

Add a new sheet called data_model (not a new column in the main sheet)
This sheet should have rows for each field showing: table_name, field_name, table_type (fact/dimension), role (pk/fk/measure/attribute), data_type, aggregation


5. Trigger Detection
The app should trigger this feature when the user says anything like:

"build a data model"
