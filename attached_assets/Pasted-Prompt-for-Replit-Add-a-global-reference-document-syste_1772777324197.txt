Update the Shared callClaude() Wrapper
Update the existing callClaude() function so that every time it is called it automatically checks whether a reference document is loaded. If one is loaded it injects the document into the messages array before the user message.
Replace the current callClaude() function with this updated version:
javascriptasync function callClaude(userPrompt, conversationHistory = [], returnJson = false, referenceDocument = null) {

  let messages = [...conversationHistory];

  // If a reference document is loaded, inject it as the first user message
  if (referenceDocument) {
    if (referenceDocument.fileType === "pdf") {
      messages = [
        {
          role: "user",
          content: [
            {
              type: "document",
              source: {
                type: "base64",
                media_type: "application/pdf",
                data: referenceDocument.content
              },
              cache_control: { type: "ephemeral" }
            },
            {
              type: "text",
              text: `This is a reference document called "${referenceDocument.filename}". Use it as background context for all your responses in this session. Always consider its contents when generating analysis, rules, classifications, or recommendations.`
            }
          ]
        },
        { role: "assistant", content: "Understood. I have read the reference document and will consider its contents in all my responses." },
        ...conversationHistory,
        { role: "user", content: userPrompt }
      ];
    } else {
      // Plain text document
      messages = [
        {
          role: "user",
          content: `This is a reference document called "${referenceDocument.filename}". Use it as background context for all your responses in this session. Always consider its contents when generating analysis, rules, classifications, or recommendations.\n\nDOCUMENT CONTENT:\n${referenceDocument.content}`
        },
        { role: "assistant", content: "Understood. I have read the reference document and will consider its contents in all my responses." },
        ...conversationHistory,
        { role: "user", content: userPrompt }
      ];
    }
  } else {
    messages = [
      ...conversationHistory,
      { role: "user", content: userPrompt }
    ];
  }

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",
      max_tokens: 1000,
      system: ZATCA_SYSTEM_PROMPT,
      messages: messages
    })
  });

  const data = await response.json();
  const text = data.content
    .filter(b => b.type === "text")
    .map(b => b.text)
    .join("");

  if (returnJson) {
    const cleaned = text.replace(/```json|```/g, "").trim();
    return JSON.parse(cleaned);
  }

  return text;
}
```

Pass `referenceDocument` from the global React state into every `callClaude()` call across the entire app.

**What the Reference Document Is For**

The reference document is intended to give Claude additional organizational context that applies to all analyses. Examples of what a user might upload:

- ZATCA internal data governance policy document
- A data dictionary or glossary PDF
- A regulatory circular from SDAIA or ZATCA
- A specific tax law or compliance framework document
- An internal data standards document

Claude should use this document as background context — not as the primary input — when generating data quality rules, classifications, business definitions, nudge strategies, insights, or any other analysis.
```
