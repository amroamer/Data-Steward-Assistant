Step 2 — Display Results On Screen
After Claude responds, show the results directly in the center panel as structured visual sections — not as a chat bubble, not as raw JSON. The user should be able to read everything on screen without downloading anything.
Display four sections one after another, each in its own card:

Section A — Diagnosis Card
A card with header: 🔍 Why is this happening?
Show inside the card:

Primary Root Cause — large bold text, ZATCA blue background highlight
Is it intentional? — Yes or No badge (red for Yes, green for No)
Secondary Root Causes — simple tag chips
Emotional Drivers — bullet list
Friction Points — bullet list
Rationale — paragraph text in italic


Section B — Taxpayer Segments Card
A card with header: 👥 Who are we dealing with?
Show a styled table inside the card with one row per segment:
| Segment Name | Archetype | Population % | Risk Level | Main Barrier | Receptiveness | Best Channel | Best Timing |
Color code the Risk Level column: Critical → red, High → orange, Medium → yellow, Low → green
Color code the Receptiveness column: High → green, Medium → yellow, Low → red

Section C — Behavioral Levers Card
A card with header: 🎯 What should we do?
Show one sub-card per lever. Each sub-card shows:

Lever name and type as a header
Target Segments — tags showing which segments this lever targets
The actual message text — displayed in a highlighted yellow box, large enough to read clearly. This is the most important thing to show — the word-for-word message the taxpayer would receive
Channel and Timing on one line
Expected Impact and Implementation Effort on one line
Priority badge — High (red), Medium (orange), Low (green)


Section D — Intervention Plan Card
A card with header: 📋 Intervention Plan
Show inside the card:

Recommended Sequence — numbered list of lever names in order
Quick Wins — green checkmark bullet list
KPIs to Track — simple bullet list
Estimated Compliance Lift — large bold number with a % sign, shown prominently


Step 3 — Summary Banner
Above all four sections show a summary banner with five stat tiles:

⚡ Quick Wins: [count]

Style the banner with ZATCA dark blue background and white text.

Step 4 — Generate Excel File
After displaying the results on screen, also generate a downloadable Excel file called nudge_report_[timestamp].xlsx using SheetJS.
The Excel file has five sheets:
Sheet 1: executive_summary

Two-column key-value layout
Show: use case, category, severity, primary root cause, is intentional, estimated lift, KPIs
Header row: ZATCA blue background, white bold text

Sheet 2: diagnosis

Two-column key-value layout
Show all diagnosis fields: primary root cause, secondary root causes, emotional drivers, friction points, rationale
Primary root cause row: ZATCA blue background white bold text to make it stand out

Sheet 3: population_segments

One row per segment
Columns: id, name, archetype, population_pct, risk_level, main_barrier, receptiveness, best_channel, best_timing
Color code risk_level and receptiveness cells same as on screen

Sheet 4: behavioral_levers

One row per lever
Columns: id, type, name, target_segments, message_text, channel, timing, expected_impact, implementation_effort, priority
message_text column: light yellow background, wrap text enabled — this is the actual nudge message
priority column: color coded High → red, Medium → orange, Low → green

Sheet 5: intervention_plan

Two-column key-value layout
Show: recommended sequence, quick wins, KPIs, estimated lift

All sheets: ZATCA blue headers, white bold text, auto-width columns, freeze header row.

Step 5 — Download Button
Show a "📥 Download Nudge Report" button in ZATCA green below the four result sections. Clicking it downloads the generated Excel file.
